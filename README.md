# Knowledge OS - Document RAG System

A modern, sleek Retrieval-Augmented Generation (RAG) platform to chat with your documents. Built using Flask, LangChain, ChromaDB, and OpenAI.

---

## 🚀 Key Features

* **Unified Context Loading**: Documents (such as resumes) are processed as single, consolidated chunks so they are read as one complete context.
* **AWS S3 Integration**: Automatically backs up all uploaded documents to an AWS S3 bucket.
* **OpenAI & ChromaDB Vector Store**: Indexes documents into a local semantic vector store using OpenAI Embeddings and ChromaDB.
* **Source Citations**: Answers generated by the AI are tagged with the specific source files used to construct the response.
* **Sleek UI**: Modern, glassmorphism dark-mode interface with toast notifications and drag-and-drop file ingestion.

---

## 🛠️ Prerequisites

* Python **>= 3.11**
* Active AWS S3 Bucket
* OpenAI API Key

---

## ⚙️ Setup & Installation

### 1. Clone & Set Up Environment

Initialize your virtual environment and install the required dependencies (using `uv` or standard pip):

```bash
# Clone the repository
git clone https://github.com/vardhan1519/Knowledge-Intelligence-System.git
cd Knowledge-Intelligence-System

# Create virtual environment and install dependencies
uv sync
```

### 2. Configure Environment Variables

Create a file named `.env` inside the `app/` folder (i.e., `app/.env`) and add your configuration credentials:

```env
OPENAI_API_KEY="your-openai-api-key"
AWS_ACCESS_KEY_ID="your-aws-access-key-id"
AWS_SECRET_ACCESS_KEY="your-aws-secret-access-key"
AWS_BUCKET_NAME="knowledge-information-system1519"
```

---

## 💻 Running the Application

Start the Flask development server:

```bash
uv run python -m app.main
```

The application will run locally and can be accessed at:
👉 **[http://localhost:8080](http://localhost:8080)**

---

## 🧹 Clearing the Vector Database

If you wish to reset or clear all indexed document context:

1. Stop the application server (`Ctrl + C`).
2. Run the following command in the project root:
   ```bash
   rm -rf vector_db
   ```
3. Restart the application. A fresh database will automatically be initialized.
